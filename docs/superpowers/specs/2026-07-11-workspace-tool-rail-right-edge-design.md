# 工作区工具栏固定最右侧设计

## 状态

已确认，待实现。

## 目标

让桌面端会话页和终端页的工作区工具栏在面板折叠与展开时都固定在窗口最右侧，并移除桌面标题栏中重复的工作区展开按钮。

## 根因

- 会话页和终端页当前都按“主内容 → 工具栏 → 工作区面板”渲染，因此面板展开后会出现在工具栏右侧。
- 标题栏仍为桌面会话/终端路由渲染工作区切换按钮，与常驻工具栏底部的展开/收起按钮功能重复。

## 方案

1. 会话页桌面布局调整为“主内容 → 工作区面板 → 工具栏”。
2. 终端页桌面布局采用同样顺序，保证两类工作区行为一致。
3. 标题栏工作区按钮仅在移动端显示；桌面端以最右侧常驻工具栏作为唯一入口。
4. 保留现有 32px 工具栏宽度、48px 工具入口高度、面板拖拽宽度、激活状态、文件/变更/指标入口及工具栏底部展开/收起按钮。
5. 不改变移动端工作区按钮、遮罩层或滑出面板行为。

## 验收标准

- 桌面会话页：工作区折叠和展开时，32px 工具栏始终贴住窗口右边缘。
- 桌面终端页：行为与会话页一致。
- 桌面标题栏不再显示工作区展开/收起按钮。
- 移动端标题栏仍保留工作区入口。
- 面板展开、收起、切换标签及拖拽调整宽度正常工作。

## 验证

- 增加结构测试，锁定会话页和终端页中“工作区面板先于工具栏”的渲染顺序。
- 增加标题栏结构测试，确认工作区按钮只受移动端条件控制。
- 运行相关 Bun 测试和前端类型检查。
