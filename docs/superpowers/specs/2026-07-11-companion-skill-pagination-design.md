# 伙伴技能列表分页设计

## 目标

为「桌面伙伴 → 伙伴 → 技能」列表提供服务端分页，默认每页 10 条，并允许用户切换为每页 20 或 50 条。

## 交互

- 状态筛选（全部、待审、已启用、已归档）由接口执行；切换筛选或每页条数时回到第 1 页。
- 分页栏仅在结果总数大于 0 时显示，显示总数、页码跳转和每页条数选择。
- 采纳、拒绝、示范教学和实时技能事件触发刷新；若当前页因条目状态变化而越界，自动回退到最后一页。
- 初次加载显示加载态；翻页或刷新保留当前卡片并降低透明度，避免列表闪烁。空结果仍显示既有空态。

## 接口契约

`GET /api/companion/companions/{companion_id}/skills` 新增可选参数：

- `status`：非空时仅返回对应生命周期状态；
- `limit`：页大小，默认 100，服务端限制为 1–500；
- `offset`：从零开始的偏移量，负数按零处理；
- `include_shared`：保持现有语义，默认包含共享技能。

响应从技能数组改为：

```json
{ "items": ["技能视图"], "total": 42 }
```

`total` 与 `items` 使用完全相同的伙伴可见范围和状态筛选条件。服务层只为当前页技能读取 `SKILL.md` 的描述，避免随技能数量增长而全量读盘。

## 范围

不调整技能审批、编辑、赠送或示范教学逻辑；不处理本次基线中与默认伙伴回退顺序有关的独立测试问题。
