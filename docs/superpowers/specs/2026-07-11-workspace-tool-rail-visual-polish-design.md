# 工作区工具栏视觉精简设计

## 状态

已确认，待实现。

## 目标

进一步精简右侧工作区工具栏：移除图标下方可见文字与激活竖杆，并让工具提示更小、更紧凑。

## 方案

1. 工具栏按钮不再显示图标下方文字；标签继续以视觉隐藏文本存在，为按钮保留可访问名称。
2. 删除 `.workspace-tool-rail__item--active::before` 激活竖杆；继续使用激活背景、边框、阴影和图标颜色表达选中状态。
3. 工具栏 Tooltip 启用 Arco `mini` 模式，并添加工具栏专属弹层类名。
4. 专属 Tooltip 使用 `11px` 字号、`16px` 行高和 `3px 6px` 内边距，仅影响工作区工具栏，不修改全局 Tooltip。
5. 保持工具栏宽度 `32px`、按钮宽度 `28px`、按钮高度 `48px`、图标尺寸 `18px`，不改变点击、徽标、状态点和键盘焦点行为。

## 验收标准

- 文件、变更、指标等按钮只显示图标，不显示下方文字。
- 激活按钮没有左侧竖杆，但仍能通过背景与颜色辨识。
- 悬浮提示比当前默认气泡更小，文字与气泡边缘更紧凑。
- 屏幕阅读器仍可识别按钮名称。
- 会话页与终端页共用同一效果，其他页面的 Tooltip 不受影响。

## 验证

- 更新工作区工具栏结构测试，验证 Tooltip 使用专属类名和 `mini`，标签采用视觉隐藏类。
- 更新样式契约测试，验证激活竖杆规则被移除、紧凑 Tooltip 尺寸存在且按钮高度保持 `48px`。
- 运行相关 Bun 测试与前端类型检查。
