# 定时任务横条列表设计

## 目标

优化定时任务列表的信息密度与可扫描性。桌面端由多列卡片改为单列横条列表；移动端继续使用现有卡片布局。用户仍可在列表中启动或暂停任务，并可点击任务进入详情。

## 范围

- 仅调整 `ScheduledTasksPage` 的任务列表展示与相关响应式布局。
- 保留搜索、创建任务、保持唤醒、空状态、加载状态和任务详情页。
- 保留现有任务数据、状态判断、启停 API 与成功/错误提示。
- 不修改后端接口、定时任务模型或调度行为。

## 桌面端设计

桌面断点使用 JSX 中的 `md:` 响应式类。桌面任务区显示为一张连续列表，每个任务占一条横向记录，记录之间使用分割线分隔。

列表包含以下列：

1. 任务标题：显示任务名称；短任务 ID 作为辅助信息保留，并允许标题截断。
2. 下次运行时间：有计划时间时显示格式化时间，无计划时间时显示 `-`。
3. 任务状态：复用 `CronStatusTag`，继续覆盖已启动、已暂停和执行错误等原有状态。
4. 执行模式：显示“新建会话”或“持续会话”，并保留可用的 Agent 标识。
5. 启停操作：非纯手动任务显示开关；纯手动任务沿用现有行为，不显示开关。

列表顶部增加与内容对齐的列标题，便于识别字段。整条记录可点击并进入任务详情。启停开关阻止点击事件冒泡，只切换任务状态，不触发页面跳转。

桌面任务行采用紧凑密度：内容区最小高度为 48px、上下 padding 为 8px，最终总高度约 64px。表头和列表容器使用透明背景，不显示四周外轮廓和圆角，只保留表头下方及任务之间的横向分隔线。

## 移动端设计

移动端沿用当前卡片样式、字段顺序和换行方式，不引入横向滚动。卡片仍显示任务名称、短 ID、状态、调度描述、下次运行时间、执行模式和启停开关。

## 响应式与可用性

- 桌面端列宽使用 CSS Grid：标题列弹性最大，下次运行时间和执行模式保持足够可读宽度，状态与开关使用紧凑宽度。
- 桌面行高约 64px，表头和列表背景透明且无外轮廓；内部横向分隔线继续保留。
- 长标题单行截断，并通过 `title` 提供完整文本。
- 横条保留 hover 边框或背景反馈，维持“可点击进入详情”的感知。
- 启停开关继续使用受控状态，后端调用失败时保留原状态并显示错误信息。

## 组件边界

桌面和移动端复用同一套任务 DOM，使用 JSX 中可被 UnoCSS 静态扫描的 `md:` 类切换横条和卡片样式。既有 `CronStatusTag`、格式化函数、详情跳转和启停回调继续复用。

## 测试与验收

- 为响应式 JSX 类、桌面密度和无外轮廓约束添加测试，再实现页面改动。
- 运行新增测试、现有定时任务页面相关测试和 TypeScript 类型检查。
- 桌面验收：任务为紧凑单列横条、背景透明、无外轮廓、带内部分割线和列标题，五类信息完整，点击记录可进入详情，开关可启停且不跳转。
- 移动验收：仍为当前卡片样式，无横向溢出，原有信息和启停操作不退化。
