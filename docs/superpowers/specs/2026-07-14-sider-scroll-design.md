# 应用侧栏溢出滚动设计

## 背景与根因

应用级左侧导航的主内容容器使用 `overflow-hidden`。当导航分组和项目的总高度超过可用窗口高度时，溢出项目既无法通过鼠标滚轮、触控板或键盘访问，也无法拖拽滚动条；底部固定的设置区会进一步遮挡内容。

## 方案取舍

1. 让整个侧栏（包括设置区）一起滚动：实现最小，但设置入口会随内容离开视区，不符合截图中底部操作区的稳定性。
2. 为导航主体建立独立纵向滚动区，并保留设置区固定：推荐。可访问全部导航项，同时维持现有信息架构和底部操作的可达性。
3. 减少项目或折叠分组：不能解决未来项目增长，也改变了用户已经熟悉的导航结构。

## 设计

仅修改 `ui/src/renderer/components/layout/Sider/index.tsx` 的主导航容器类名：保留其在外层 flex 布局中的高度约束（`flex-1 min-h-0`），将裁切改为纵向自动滚动，并禁止横向滚动。底部 `SiderFooter` 继续是独立的 `shrink-0` 区域。

结果：

- 内容未溢出时，视觉保持不变。
- 内容溢出时，鼠标滚轮、触控板和可拖拽滚动条均可访问全部导航项。
- 聚焦导航项时，浏览器会将其滚入可见区；固定设置区不会遮挡主导航内容。
- 不新增状态、组件、依赖、文案或业务逻辑，也不更改会话二级侧栏。

## 验证

1. 添加结构回归测试，断言主导航容器同时具备 `overflow-y-auto`、`overflow-x-hidden` 与现有的弹性高度约束，且底部区域仍为 `shrink-0`。
2. 先运行该测试确认现状失败，再实施最小 className 修改并确认测试通过。
3. 运行 UI 类型检查，确保 TSX 修改未引入编译错误。
