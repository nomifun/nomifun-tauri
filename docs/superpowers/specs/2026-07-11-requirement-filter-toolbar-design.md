# 需求筛选工具栏视觉优化设计

日期：2026-07-11
状态：已确认，待实施

## 背景与目标

需求平台当前使用三个常规输入控件呈现标签、状态、搜索，并在右侧单独展示排序字段与升降序按钮。控件视觉较重、占用两行，与参考图中的轻量工具栏风格不一致。

本次仅重做筛选工具栏的展示与局部交互：标签、状态、排序统一为“图标 + 功能文字 + 当前选中内容”的紧凑入口；搜索默认显示“图标 + 搜索”，点击后展开输入框。现有查询参数、筛选结果、分页重置、看板行为和批量删除操作保持不变。

## 方案选择

采用自定义触发器配合现有 Arco 下拉能力：触发器负责参考图风格的展示，原有选项和值变更回调继续负责业务行为。

未采用的方案：

- 仅压缩现有 `Select`：改动最小，但仍保留明显的输入框外观，无法达到参考图的轻量效果。
- 使用统一筛选弹窗：视觉集中，但增加操作层级，不适合频繁切换单个筛选项。

## 界面与交互

### 工具栏布局

工具栏为单个可换行的水平容器，依次放置标签、状态、排序和搜索入口。入口不显示常驻边框，默认背景透明，悬停、键盘聚焦或弹层打开时显示轻微的主题色背景。颜色、圆角和文字均使用现有主题变量，兼容明暗主题。

### 标签

- 未选择：显示“标签图标 + 标签”。
- 已选择：显示“标签图标 + 标签 + 当前标签名”。
- 点击后打开标签下拉菜单，选项仍显示名称及完成数/总数。
- 已选状态可在下拉中清除，清除后回到未选择样式。

### 状态

- 未选择：显示“筛选图标 + 状态”。
- 已选择：显示“筛选图标 + 状态 + 本地化状态名称”。
- 点击后打开现有六种状态选项，并支持清除。

### 排序

- 默认顺序：显示“排序图标 + 排序”。
- 已选择字段：显示“排序图标 + 排序 + 当前字段名称”。
- 点击后在同一个下拉内容中选择默认顺序、ID、创建时间、更新时间或状态。
- 选择具体字段时，弹层同时提供升序/降序切换；选择默认顺序时不显示或禁用方向切换。
- 当前升降序通过方向图标和选中态表达，不再在工具栏中保留独立的大按钮。

### 搜索

- 搜索词为空且未激活时，仅显示“搜索图标 + 搜索”。
- 点击入口后原位展开搜索输入框并自动聚焦。
- 输入继续调用现有 `onSearchChange`，搜索行为和分页重置逻辑不变。
- 按 Escape 或在空内容时失焦，输入框收回为入口。
- 搜索词非空时保持输入框展开，避免隐藏当前生效的筛选条件；清空并失焦后才收回。

### 响应式行为

窄宽度下入口允许整体换行，不压缩文字到不可读。展开的搜索框使用有限宽度和可收缩上限；不会覆盖新建需求按钮或批量操作条。

## 组件边界与数据流

改动限定在 `RequirementFilters.tsx` 及其必要的同目录测试：

- `WorkspacePage/index.tsx` 继续拥有 `tag`、`status`、`search`、`orderBy`、`order` 状态。
- `RequirementFilters` 继续通过现有 props 接收状态并发出变更回调。
- 本地新增的仅是搜索入口是否展开、弹层呈现等纯 UI 状态。
- 不修改 `useRequirements`、IPC、后端查询或服务端排序。
- 现有批量删除操作条结构与行为保持不变。

## 可访问性

- 所有触发器使用真实按钮或等价的可聚焦控件，支持 Enter/Space 打开。
- 图标为装饰时对辅助技术隐藏，功能名称和当前值作为可读文本保留。
- 搜索输入框展开后自动聚焦，Escape 可退出。
- 悬停、聚焦与选中状态不能只依赖颜色表达；当前值和方向均有文字或图标提示。

## 测试与验证

- 添加组件行为测试，覆盖：未选/已选文案、标签与状态回调、排序字段和方向回调、搜索点击展开、空内容失焦收起、非空搜索保持展开。
- 运行 UI 类型检查，确认 Arco 组件和图标类型正确。
- 运行相关测试与 UI 构建。
- 人工在需求平台列表和看板视图检查明暗主题、窄宽度换行、键盘操作、筛选结果和分页重置。

## 范围之外

- 不改变标签、状态、排序或搜索的业务语义。
- 不新增筛选字段，不修改筛选 URL 持久化策略。
- 不调整需求卡片、看板列、分页或批量操作条的视觉样式。
- 不修改服务端接口、查询或排序实现。
